# ADR-001: Padrão de Persistência de Dados em Markdown

**Status**: Aceito  
**Data**: 2024 (revisado 2026-06-16)

---

## Contexto

O script original (Orion Design) persistia dados em arquivos de texto plano acessados via `grep`/`awk`. Para tornar o ambiente amigável para o agente Claude e para humanos, precisamos de um formato auto-documentado, legível e parseável sem ferramentas externas.

---

## Decisão

Toda persistência de dados gerada pelas deploy skills usa arquivos Markdown (`.md`) em `/root/dados_vps/`.

### Convenção de caminho

```
/root/dados_vps/<skill-name>.md
```

Exemplos:
- `/root/dados_vps/traefik.md`
- `/root/dados_vps/infra-postgres.md`
- `/root/dados_vps/app-chatwoot.md`

### Camada de abstração: `lib-persistence.sh`

Toda deploy skill **deve** carregar a biblioteca antes de qualquer operação de leitura/escrita:

```bash
SKILL_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
source "$SKILL_DIR/../00-core/lib-persistence.sh"
```

Funções disponíveis:

| Função | Uso |
|--------|-----|
| `save_data "<skill>" "<conteúdo markdown>"` | Escreve/sobrescreve o arquivo de persistência |
| `read_data "<skill>" "<chave>"` | Lê valor de uma chave do arquivo |
| `append_data "<skill>" "<conteúdo>"` | Acrescenta conteúdo sem sobrescrever |

### O que persistir

✅ Permitido:
- URLs de acesso
- Nome de stacks, redes, volumes
- Usuários não-sensíveis
- Status de instalação
- Datas de deploy
- Referência a arquivo YAML gerado (ex: `/root/chatwoot.yaml`)

❌ Proibido (ver ADR-002):
- Senhas
- Tokens de API
- Chaves privadas
- Hashes de segredos

### Conteúdo do `metadata.json`

Cada deploy skill declara seu caminho de persistência:

```json
{
  "persistence_path": "/root/dados_vps/app-minhaskill.md"
}
```

---

## Consequências

**Prós**:
- Leitura humana facilitada — qualquer editor abre
- Parsing nativo pelo Claude sem ferramentas externas
- Claude Code Skills podem injetar dados dinamicamente: `` !`cat /root/dados_vps/traefik.md` ``
- Auditável em shell: `cat /root/dados_vps/*.md`

**Contras**:
- Requer a camada `lib-persistence.sh` para escrita padronizada
- Não é transacional — sobrescrita concorrente pode corromper

---

## Alternativas Rejeitadas

- **SQLite**: overhead desnecessário para volume de dados esperado
- **Texto plano com delimitadores**: não legível por Claude sem parsing customizado
- **Variáveis de ambiente**: não sobrevivem à reinicialização do container/host
