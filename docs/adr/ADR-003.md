# ADR-003: Padrão de Claude Code Skills para Orquestração

**Status**: Aceito  
**Data**: 2026-06-16

---

## Contexto

O Claude Code possui um sistema nativo de extensão via arquivos `SKILL.md` que permitem ensinar ao agente procedimentos reutilizáveis, invocáveis via `/nome` ou automaticamente quando relevante. Este projeto tem dois tipos de "skill" (ver CLAUDE.md — Glossário) e precisamos de convenções claras para as Claude Code Skills que orquestram as deploy skills do ecossistema Orion.

---

## Decisão

### 1. Localização

```
/root/setup-skills/.claude/skills/<nome>/SKILL.md
```

Para skills pessoais (todos os projetos):
```
~/.claude/skills/<nome>/SKILL.md
```

### 2. Estrutura mínima obrigatória

```
.claude/skills/<nome>/
├── SKILL.md          # obrigatório — frontmatter + instruções
└── scripts/          # opcional — scripts auxiliares
```

### 3. Frontmatter canônico

```yaml
---
description: |
  Uma linha clara do que faz. CASO DE USO PRINCIPAL PRIMEIRO.
  Frases gatilho: quando usuário pede X, Y ou Z.
disable-model-invocation: true   # obrigatório para skills com side-effects
allowed-tools: Bash(docker *) Read
---
```

**Campos obrigatórios**:
- `description`: Claude usa para decidir quando carregar. Truncado em 1536 chars — caso principal primeiro.

**Campos recomendados por tipo**:

| Tipo de skill | Campos adicionais |
|--------------|-------------------|
| Deploy/instalação | `disable-model-invocation: true` |
| Auditoria/diagnóstico | `context: fork`, `agent: Explore` |
| Background knowledge | `user-invocable: false` |
| Workflow manual | `disable-model-invocation: true`, `argument-hint` |

### 4. Injeção de contexto dinâmico

Use `` !`cmd` `` para injetar estado real ANTES de Claude processar:

```markdown
## Estado atual do ecossistema
!`cat /root/dados_vps/traefik.md 2>/dev/null || echo "traefik não instalado"`
!`docker service ls 2>/dev/null | head -30`
!`ls /root/setup-skills/skills/`
```

**Regras**:
- `!` deve estar no início da linha (ou após whitespace)
- Executa uma vez — output não é re-processado
- Nunca injetar segredos via `!`cmd`` (ADR-002)

### 5. Nomenclatura de skills por categoria

| Prefixo | Categoria | Exemplo |
|---------|-----------|---------|
| `deploy-` | Instala uma stack | `deploy-chatwoot` |
| `audit-` | Audita skills/stacks | `audit-skills` |
| `diagnose-` | Diagnostica problemas | `diagnose-stack` |
| `status-` | Reporta estado | `status-ecossistema` |
| `migrate-` | Migração/atualização | `migrate-evolution` |

### 6. Skills com subagent isolado

Para auditoria e diagnóstico que não devem interferir na conversa principal:

```yaml
---
description: Audita todas as deploy skills e reporta conformidades.
context: fork
agent: Explore
allowed-tools: Read Bash(cat *) Bash(ls *) Bash(find *)
---

## Tarefa
!`find /root/setup-skills/skills -name "metadata.json" | sort`

Auditar cada skill acima verificando:
1. Presença de metadata.json, run.sh, README.md
2. Campos obrigatórios no metadata.json
3. Conformidade com ADR-001 (save_data) e ADR-002 (sem segredos)

Retornar relatório estruturado em markdown.
```

### 7. Tamanho e arquivos de suporte

- `SKILL.md` deve ter **menos de 500 linhas**
- Material extenso vai em arquivos auxiliares referenciados no `SKILL.md`:

```markdown
## Referências
- Para lista completa de stacks: [../skills-catalog.md](../skills-catalog.md)
- Para convenções de metadata: [../../docs/adr/ADR-003.md](../../docs/adr/ADR-003.md)
```

### 8. Substituições disponíveis

| Variável | Uso neste projeto |
|----------|-------------------|
| `$ARGUMENTS` | Nome da skill ou stack passada pelo usuário |
| `$0`, `$1` | Args posicionais (ex: `/deploy-skill chatwoot staging`) |
| `${CLAUDE_SKILL_DIR}` | Referenciar scripts bundled com a skill |
| `${CLAUDE_SESSION_ID}` | Log de auditoria por sessão |

---

## Consequências

**Prós**:
- Skills de orquestração reutilizáveis e versionadas no repositório
- Injeção dinâmica elimina necessidade de descrever estado — Claude vê realidade
- `context: fork` para auditoria: isolamento total, sem poluir contexto da conversa
- Live change detection: editar SKILL.md reflete imediatamente sem reiniciar Claude Code

**Contras**:
- Conteúdo da SKILL.md invocada fica em contexto até fim da sessão (custo de tokens)
- Skills com `context: fork` não têm acesso ao histórico da conversa
- Criar pasta de skills nova DURANTE sessão exige reiniciar Claude Code

---

## Alternativas Rejeitadas

- **Tudo no CLAUDE.md**: CLAUDE.md carrega sempre, skills carregam só quando usadas — skills são mais baratas para conteúdo longo/condicional
- **Scripts bash standalone**: sem integração com o fluxo de chat do Claude Code; não aproveitam injeção dinâmica nem subagents
- **`.claude/commands/` (formato legado)**: funciona, mas skills têm superset de features (arquivos de suporte, `context: fork`, `paths:`)
